vLLM Model Parallelism Decision Tree

How to load a model across multiple GPUs

START
1. What type of model?
DENSE MODEL
(No MoE)
Does the model fit on one GPU with enough memory left for KV cache?
Use single GPU
TP = 1, PP = 1
Does the model fit within one GPU node?
Use the smallest TP size that fits
TP > 1, PP = 1
Compare options:
  • TP
  • PP
  • TP + PP
Prefer:
  • TP within each node
  • PP across nodes
MIXTURE-OF-EXPERTS MODEL
(MoE)
Do the shared/attention layers fit on one GPU?
Start with TP = 1
Use the smallest TP size that fits
Step A: Size attention & shared layers ✓
Step B: Place expert weights
Do expert weights fit acceptably without EP?
EP may not be necessary
Enable EP
EP size = TP × DP
Step C: Choose DP (MoE attention lanes)
How many attention lanes do you need?
  • More request throughput
  • More concurrency
  • More independent KV caches
  • Wider expert mesh (EP = TP × DP)

Select DP size accordingly
Step D: Does the complete MoE model fit?
Does the full model fit in memory with TP, DP, and EP?
Use TP + DP + EP
Add PP to divide the layer stack
2. Validate the working set (KV cache)
Is there enough KV-cache capacity for your target context length and concurrency?
Proceed to performance check
Address KV-cache memory pressure (choose one or more):

Reduce weight memory

  • Lower-precision weights (FP8/INT4)
  • Adjust TP/PP layout

Reduce KV-cache memory

  • Use FP8 KV cache (where supported)

KV cache duplicated?

Is TP > effective # of KV heads?

→ Consider DCP (Context Parallelism)

Context very large?

  • Evaluate DCP
  • KV-cache offload
  • Lower concurrency / context limits
3. Final performance check
Does the selected parallelism topology meet latency goals?

TTFT too high (prefill issue)

  • Inspect prefill saturation
  • Long prompts?
  • Tune chunked prefill
  • Check PP stage balance

TPOT too high (decode issue)

  • Inspect TP communication
  • Inspect EP all-to-all communication
  • Inspect expert imbalance
  • Check decode saturation

Latency acceptable
